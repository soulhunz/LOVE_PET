# LOVE_PET (MyDesktopPet) — เอกสารสรุปทั้งโปรแกรม

> สัตว์เลี้ยงเดสก์ท็อป (Desktop Pet) บน Windows — ตัวละครอนิเมชันเดินไปมาบนหน้าจอจริง
> เลี้ยงดู / ต่อสู้ / ผสมพันธุ์ / สะสม พร้อมระบบเกมครบวงจร เขียนด้วย Python + Tkinter ล้วน

---

## 1. ภาพรวม

LOVE_PET เป็นโปรแกรม "สัตว์เลี้ยงบนหน้าจอ" ที่หน้าต่างโปร่งใสครอบทุกจอภาพ ตัวละครยืนอยู่บน
taskbar เดินเล่น โต้ตอบเมื่อคลิก ลากย้ายตำแหน่งได้ และมีระบบเกมเลี้ยงสัตว์/ต่อสู้/สะสม
ครบทั้งวงจร โดย**ไม่ต้องมีไฟล์ภาพก็เล่นได้** (โปรแกรมวาดตัวละครสำรองด้วยโค้ดเอง)

- **แพลตฟอร์ม:** Windows (ใช้ `winsound`, Win32 API ผ่าน `ctypes`, หน้าต่างโปร่งใสแบบ `-transparentcolor`)
- **ภาษา/ไลบรารี:** Python 3 + Tkinter (ทั้งหมดเป็น stdlib — ไม่ต้องติดตั้ง dependency เพิ่ม)
- **การแจกจ่าย:** รันเป็นสคริปต์ `python main.py` หรือแพ็กเป็น `.exe` ไฟล์เดียวด้วย PyInstaller
- **ภาษาในเกม:** ภาษาไทย

### วิธีรันแบบเร็ว

| วิธี | คำสั่ง / ไฟล์ | หมายเหตุ |
|------|--------------|----------|
| รันตรง ๆ | `python main.py` | กด `Esc` เพื่อออก |
| เล่นแบบไม่มีหน้าต่างดำ | ดับเบิลคลิก `play.bat` | ใช้ `pythonw` |
| ดูข้อผิดพลาด | `run_debug.bat` | รันด้วย `python` มี console |
| ติดตั้ง + auto-start | `install.bat` | สร้าง shortcut + เปิดตอนบูต |

## 2. โครงสร้างไฟล์

```
LOVE_PET/
├── main.py            จุดเริ่มโปรแกรม + ดักจับ crash → error.log + กล่องข้อความ
├── game.py            หัวใจของเกม (คลาส World) — เกมลูป, ต่อสู้, UI, ระบบทั้งหมด (~4400 บรรทัด)
├── entities.py        คลาสวัตถุบนจอ: Entity, Pet, Monster, Food (ข้อมูล + การเรนเดอร์)
├── config.py          ค่าตั้งทั้งหมด (สมดุลเกม, สเตตัส, สกิล, ราคา ฯลฯ) — แก้ที่นี่ที่เดียว
├── assets.py          โหลดสไปรต์จากไฟล์ + วาดตัวละครสำรอง + จัดการโฟลเดอร์ตัวละคร/มอนสเตอร์
├── paths.py           จัดการ path ให้ทำงานทั้งตอนรันสคริปต์และตอนเป็น .exe
├── save.py            อ่าน/เขียนไฟล์เซฟ (save.json) แบบ atomic
├── sound.py           เสียง 8-bit สังเคราะห์เองด้วยโค้ด (winsound)
│
├── assets/            ไฟล์ภาพที่มากับโปรแกรม (อ่านอย่างเดียว) — มอนสเตอร์/อาหาร/ไข่
├── characters/        ตัวละครตัวอย่าง (แต่ละโฟลเดอร์ = 1 ตัวละคร มี pet.json + pet_*.png)
│   ├── ก้อนปุย/
│   └── หิมะขนปุย/
│
├── MyDesktopPet.spec  สคริปต์ PyInstaller (datas รวม assets/ + characters/)
└── *.bat / *.vbs      เครื่องมือ Windows (ดูตาราง §8)
```

> **ข้อมูลผู้ใช้** (เขียนได้) เก็บที่ `%APPDATA%\MyDesktopPet\` ไม่ใช่ในโฟลเดอร์โปรแกรม:
> `save.json`, โฟลเดอร์ `characters/` และ `monsters/` ที่ผู้ใช้เพิ่มเอง
> (โครงสร้างนี้ทำให้ติดตั้งใน Program Files หรือเป็น `.exe` แล้วยังเซฟได้)

## 3. โมดูลแต่ละไฟล์

### `main.py` — จุดเริ่มต้น
- ตั้ง working directory เป็น "โฟลเดอร์โปรแกรม" เสมอ (ทั้งตอนรันสคริปต์และตอนเป็น `.exe`)
- เรียก `World().run()`
- `_report_crash()` — ถ้า error: เขียน `error.log` + เด้งกล่องข้อความ (เห็นได้แม้รันด้วย `pythonw` ที่ไม่มี console)

### `paths.py` — เส้นทางไฟล์
- `resource_path(*parts)` — ไฟล์ที่มากับโปรแกรม (อ่านอย่างเดียว); รองรับ `sys._MEIPASS` ของ PyInstaller
- `data_path(*parts)` — ไฟล์ข้อมูลผู้ใช้ (เขียนได้) ที่ `%APPDATA%\MyDesktopPet`

### `save.py` — บันทึก/โหลด
- เซฟที่ `%APPDATA%\MyDesktopPet\save.json`
- เขียนแบบ atomic (เขียนไฟล์ `.tmp` ก่อน แล้ว `os.replace`) กันไฟล์พังถ้าโปรแกรมดับกลางคัน
- ย้ายเซฟเก่าที่เคยอยู่ข้างโปรแกรมมาที่ใหม่อัตโนมัติ (ครั้งแรกที่อัปเกรด)

### `sound.py` — เสียง
- สังเคราะห์คลื่นเสียง 8-bit เอง (square/saw/sine) แล้วเล่นผ่าน `winsound` ในเธรดแยก (ไม่บล็อก UI)
- เหตุการณ์: `eat`, `attack`, `hurt`, `win`, `levelup`, `boss`, `click`, `pet`
- เปิด/ปิดได้ด้วย `set_enabled()`; บนระบบที่ไม่ใช่ Windows จะเงียบ

### `assets.py` — ภาพและตัวละคร
- **`Animation`** — ลำดับเฟรมภาพ 1 ชุด สร้างเฟรมกลับด้าน (mirror) เมื่อต้องใช้ครั้งแรก
- **โหลดสไปรต์:** `load_sprite()` รองรับ
  - `.gif` หลายเฟรม = อนิเมชัน
  - `.png` ภาพนิ่ง / sprite sheet (เฟรมจัตุรัสตัดอัตโนมัติ) / ระบุจำนวนเฟรมในชื่อไฟล์ (`_strip4`, `_4frames`)
- **ลบพื้นหลังอัตโนมัติ** (`AUTO_REMOVE_BG`): PNG ที่มี alpha คงไว้; ภาพพื้นทึบใช้ flood-fill จากขอบ + ลบ halo (ขอบรุ่งริ่ง)
- **ตัวสำรอง** (`build_fallback`): วาดตัว blob เป็นพิกเซลเอง (pet/monster/food) เมื่อไม่มีไฟล์
- **จัดการตัวละคร/มอนสเตอร์:** สร้าง/ลบ/เปลี่ยนชื่อ/อ่าน-เขียน `pet.json` (likes, dislikes, rarity, display)
  - ตัวอย่างที่มากับโปรแกรมถูก seed ไปที่ `%APPDATA%` ครั้งแรกที่เปิด

### `entities.py` — วัตถุบนจอ
- **`Entity`** (ฐาน): ตำแหน่ง (x,y = กึ่งกลาง), เฟรมอนิเมชัน, การหันซ้าย-ขวา (flip), render บน canvas
  - `current_anim()` มีระบบ fallback ของสถานะ (เช่น `dead`→`hurt`→`idle`)
- **`Pet`**: สเตตัส + ค่าต่อสู้ทั้งหมด พร้อมเมธอดคำนวณ:
  - `max_hp()`, `attack()` = base × (เลเวล + ฝึกฝน) × นิสัย × รีบอร์น × ความหายาก
  - `crit_chance()`, `dodge_chance()`, `lifesteal_pct()`, `ult_damage()` จากแต้มบิลด์
  - `xp_to_next()`, `alive_ratio()`, `xp_ratio()`
- **`Monster`**: hp/atk, is_boss, stun/poison ttl
- **`Food`**: อาหารชิ้นเดียวบนจอ

## 4. `game.py` — คลาส World (เกมเอนจิน)

ไฟล์ใหญ่สุด รวมทุกระบบไว้ในคลาสเดียว แบ่งหน้าที่ได้เป็นกลุ่ม:

### 4.1 การตั้งค่าหน้าต่าง & หลายจอ
- `_enable_dpi_awareness()` — รองรับ High-DPI
- `_screen_metrics()`, `_enumerate_monitors()` (ผ่าน `EnumDisplayMonitors`) — ครอบทุกจอ, รู้ขอบ taskbar ของแต่ละจอ
- หน้าต่างไร้ขอบ (`overrideredirect`) + โปร่งใส (`-transparentcolor`) + อยู่บนสุด (`-topmost`)

### 4.2 เกมลูป
- `run()` → `_anim()` (วนเฟรมอนิเมชัน ~ทุก `ANIM_STEP_MS`) และ `_tick()` (อัปเดตเกม ~30 FPS)
- `_tick()` เรียก: decay สเตตัส, พฤติกรรมน้อง (wander/stay/sleep/eat), การต่อสู้, สุ่มเกิดมอน, วาด HUD/หลอด/ออร่า, ตรวจหิว, ซ่อนตอนมีโปรแกรม fullscreen

### 4.3 น้องหลายตัว (Pet management)
- `self.pets[]` + `self.active` (น้องที่กำลังดูแล) — property `self.pet`, `self.character`
- ช่องน้องบนจอ `PET_SLOTS_BASE..MAX` (ซื้อเพิ่มด้วยเหรียญ), กล่องเก็บน้อง `MAX_STORED`
- `_build_pet()` / `_pet_to_data()` — สร้างน้องจากเซฟ / แปลงน้องเป็น dict
- เก็บ/ปล่อย/รับเลี้ยง: `_store_pet`, `_retrieve_pet`, `_release_pet`, `_adopt_pet`

### 4.4 พฤติกรรม & อนิเมชัน
- `_update_wander` (เดินเล่นสุ่ม), `_update_stay`, `_update_sleep`, `_update_eating`, `_update_action` (อาบน้ำ/ลูบหัว), `_update_dead`
- `_speed()` — ความเร็วรวมผลจากนิสัย/ฝึก/ง่วง
- การหันหน้าและ flip ภาพอัตโนมัติเมื่อเดินสวนทาง

### 4.5 การต่อสู้ (Combat)
- `_update_combat()` — หัวใจการต่อสู้: เล็งเป้า, เข้าใกล้, ตี, คูลดาวน์, ทีมบัฟ
- `_pet_hit_monster()` — คริ/หลบ/ดูดเลือด/พิษ/แช่แข็ง/ทอนกำลัง + สกิลติดตัว + บิลด์
- `_fire_ultimate()` — ท่าไม้ตาย (เกจเดือดเต็ม → ดาเมจหนัก + สตันมอน)
- `_combat_win()` — รับเหรียญ/XP, ดรอปไข่, นับเวฟ/บอส, เควส
- `_kill_pet()`, `_update_dead()` — น้องสลบ (KO) → ชุบชีวิตได้
- ระบบเวฟ: กำจัดมอน `WAVE_LENGTH` ตัว → ตัวสุดท้าย = บอส → เวฟใหม่มอนเก่งขึ้น

### 4.6 ระบบเลี้ยงดู
- ให้อาหาร (`feed`, `_auto_feed`, `_pick_food_for`) — มีชนิดอาหาร + ความชอบเฉพาะตัว
- ลูบหัว (`pet_react`), นอน (`sleep_toggle`/`_start_sleep`/`_wake`), อาบน้ำ (`bathe`)
- `_decay_stats()` — ลดความอิ่ม/สุข/พลังงาน/ความสะอาด ตามเวลา + ระบบป่วย + สายสัมพันธ์
- `_apply_offline_decay()` — เดินเวลาต่อตอนปิดโปรแกรม (มีเพดาน `OFFLINE_MAX_HOURS`)
- แจ้งเตือนหิวเป็น Windows toast (`_notify_hungry`, `notify`)

### 4.7 ระบบความก้าวหน้า
- เลเวล/XP (`add_xp`), แต้มสกิล + บิลด์ (`_spend_sp`), รีบอร์น (`_rebirth`)
- ฝึกฝน atk/hp/speed (`_train`)
- นิสัย (Trait), สกิลติดตัว (Skill), เพศ, ความหายาก (Rarity)
- อายุ/ช่วงวัย (`_age_days`, `_age_stage`), วันเกิด (`_check_birthday`)

### 4.8 ผสมพันธุ์ / ไข่ / ฟัก
- `_breed()` — ต้องคนละเพศ + สายสัมพันธ์/อายุถึงเกณฑ์
- `_make_egg`, `_hatch_egg` (สืบทอดนิสัย/สกิล/ความชอบ), `_sell_egg`
- ช่องเก็บไข่ซื้อเพิ่มได้, ฟักนับเวลาจริงแม้ปิดโปรแกรม (`HATCH_SECONDS`)

### 4.9 เศรษฐกิจ & กิจกรรมรายวัน
- เหรียญ (`add_coins`), เช็คอินรายวัน (streak), เควสรายวัน, ความสำเร็จ (Achievement)
- ร้านค้า + กระเป๋าของ (`_show_shop_window`, `_use_item`, `_show_bag_window`)
- พาผจญภัย idle (`_send_adventure`, `_return_adventure`) — เดินเวลาแม้ปิดเกม
- มินิเกมเป่ายิงฉุบ (`_show_rps_window`), สอน/แสดงทริค (`_teach_trick`, `_perform_trick`)

### 4.10 UI / หน้าต่าง
- HUD มุมขวาล่าง (ปุ่ม: ต่อสู้, สำเร็จ, น้อง, กระเป๋า, ผจญภัย, ตั้งค่า, เช็คอิน, ร้าน, เควส) — ยุบ/กาง/ลากได้
- หน้าต่างย่อยจำนวนมาก: เมนูดูแลน้อง, สถานะ, ฝึกฝน, บิลด์, ผสมพันธุ์, รับเลี้ยง, ฟักไข่, จัดการตัวละคร/มอนสเตอร์, ตั้งค่า, เกี่ยวกับ
- หลอดสเตตัสลอยเหนือหัว, ออร่าสกิล, สีหน้าอารมณ์, ข้อความลอย, บับเบิลคำพูด
- ตั้งค่า: เสียงเปิด/ปิด, auto-start ตอนบูต, เปลี่ยนตัวละคร, อัปเดตโปรแกรม

### 4.11 อินพุต
- คลิกซ้าย = ลูบหัว / ลาก = ย้ายตำแหน่ง (`on_press`/`on_drag`/`on_release`)
- คลิกขวาที่น้อง = เมนูดูแล (`_show_pet_menu`)
- `Esc` = ออก

## 5. `config.py` — ระบบเกมโดยสรุป (ค่าหลัก)

| ระบบ | ค่าสำคัญ |
|------|----------|
| **สเตตัสพื้นฐาน** | HP 100, ความอิ่ม/สุข/พลังงาน/ความสะอาด (0–100) ลดตามเวลา |
| **ป่วย (Sickness)** | ถูกละเลย → ป่วย; ความสุขลดเร็ว ×2, เสียเลือด; ใช้ยา/ดูแลดีหายเอง |
| **สายสัมพันธ์ (Affection)** | 0–100 ค่าระยะยาว สูงแล้วลด decay; ปลดล็อกทริค + ผสมพันธุ์ |
| **เลี้ยงหลายตัว** | ช่องบนจอ 4→10 (ซื้อเพิ่ม), กล่องเก็บ 20, ระบบ social (อยู่ใกล้กันได้สุข) |
| **ผสมพันธุ์/ไข่** | ต้องคนละเพศ + affection ≥40 + อายุ ≥3 วัน; ฟัก ~15 นาที (เวลาจริง) |
| **ความหายาก (Rarity)** | common/rare/epic/legendary → ตัวคูณ ATK/HP + ราคาขาย |
| **เพศ** | ♂/♀ สุ่มตอนเกิด/ฟัก |
| **นิสัย (Trait)** | ขยัน/นักสู้/ตะกละ/ร่าเริง/ว่องไว — ผลต่อ decay/atk/speed/สุข |
| **สกิลติดตัว (Skill)** | 12 สกิล: โจมตี/บัฟ/ดีบัฟ (berserk, ironhide, regen, venom, frost ฯลฯ) |
| **บิลด์ (แต้มสกิล)** | crit / dodge / lifesteal / พลังท่าไม้ตาย (ลงเองตอนเลเวลอัพ) |
| **เลเวล/XP** | Lv.1–50, HP/ATK เพิ่มต่อเลเวล, รีบอร์นได้ที่ Lv.MAX (+15% ถาวร ×10) |
| **ท่าไม้ตาย (Ultimate)** | เกจเดือดเต็ม → ดาเมจ ×2.5+ + สตันมอน |
| **เวฟ/บอส** | 5 ตัว/เวฟ ตัวที่ 5 = บอส (เลือด ×3, ATK ×1.8, ตัวใหญ่ ×2, XP ×3) |
| **เศรษฐกิจ** | เหรียญจากชนะมอน/บอส/ให้อาหาร + เช็คอิน + เควส + ผจญภัย |
| **อาหาร** | 5 ชนิด (เนื้อ/ปลา/ผัก/ผลไม้/ขนม) + ความชอบเฉพาะตัว |
| **อายุ/ช่วงวัย** | ลูกน้อย→วัยเด็ก→วัยรุ่น→โตเต็มวัย→ผู้ใหญ่ (นับเป็นวันจริง) + โบนัสวันเกิด |
| **กลางวัน/กลางคืน** | ตามเวลาจริง (20:00–06:00) กลางคืนพลังงานลดเร็ว |
| **เวอร์ชันเซฟ** | `SAVE_VERSION = 3` |

> **ปรับสมดุลเกม:** แก้ค่าใน `config.py` ที่เดียว — มีคอมเมนต์ภาษาไทยอธิบายทุกค่า

## 6. การไหลของข้อมูล (Data flow)

```
เปิดโปรแกรม → main.py → World.__init__()
  ├── สร้างหน้าต่างโปร่งใสครอบทุกจอ
  ├── assets โหลดสไปรต์ (มอนสเตอร์/อาหาร) จาก assets/
  ├── save.load() → _load_progress() → สร้าง pets[] จากเซฟ
  │     └── _apply_offline_decay() เดินเวลาที่หายไป
  └── run() → loop: _anim() + _tick()
                       └── เปลี่ยนแปลง state → _save_progress() → save.json (atomic)
```

- **ไฟล์เซฟ** `%APPDATA%\MyDesktopPet\save.json` เก็บ: น้องทั้งหมด (สเตตัส/เลเวล/บิลด์/นิสัย),
  เหรียญ, ไข่, กระเป๋าของ, เควส, เช็คอิน, ความสำเร็จ lifetime, ตัวละครที่เลือก, เวลาเซฟล่าสุด
- **ตัวละคร** = โฟลเดอร์ `characters/<ชื่อ>/` มี `pet.json` (likes/dislikes/rarity/display) + `pet_<state>.png`

## 9. การ Build เป็น `.exe`

```bat
build.bat
```
หรือตรง ๆ:
```bat
py -m PyInstaller --noconfirm --onefile --windowed --name MyDesktopPet ^
  --add-data "assets;assets" --add-data "characters;characters" main.py
```
ผลลัพธ์: `dist\MyDesktopPet.exe` (ไฟล์เดียว ไม่ต้องมี Python) — `paths.py` จัดการให้ assets
อ่านจาก bundle และเซฟไปที่ `%APPDATA%` โดยอัตโนมัติ

## 10. หมายเหตุทางเทคนิค

- **ทั้งหมดเป็น stdlib** — `tkinter`, `winsound`, `ctypes`, `json`, `glob`, `shutil` ฯลฯ ไม่มี dependency ภายนอก
- **เฉพาะ Windows** — หน้าต่างโปร่งใสคลิกทะลุ, `winsound`, Win32 monitor API; บนระบบอื่นจะรันไม่ครบ/เงียบ
- **ทนต่อความผิดพลาด** — เซฟ atomic, ดักจับ crash เขียน log, ภาพ/เสียงที่หายใช้ของสำรอง
- **เวลาเดินจริง** — decay, ฟักไข่, ผจญภัย คำนวณจากเวลาจริงแม้ปิดโปรแกรม (มีเพดานกันลงโทษหนัก)
- **โค้ดมีคอมเมนต์ภาษาไทยละเอียด** อธิบายเหตุผลการออกแบบในเกือบทุกฟังก์ชัน
